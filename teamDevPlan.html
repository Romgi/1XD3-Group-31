The group we are designing the website for is McMaster Concert Band and the name of the app is ConcertHelper.

The Purpose

The purpose of this app is to provide both a public-facing information hub and an internal platform for music distribution and band resources for the McMaster Concert Band. We have chosen to build this particular product because the McMaster Concert Band faces two big problems: current band members do not have easy online access to their music parts, reference recordings, and season information, and concertgoers do not have easy access to information about the ensemble, upcoming concerts, past performances, and recordings. The goal of our website is to provide ease of access to relevant information for concertgoers and current band members.

Data Description

The first database table will store the reference recordings for band members sorted by part in the ensemble. We need a database to store all reference recordings for each individual part. This information will be for band members.

  • recording_id(VARCHAR) - PK
  • part_name(VARCHAR)
  • recording_url(VARCHAR)

The second database table will store the music parts for each instrument based on the current concert being played. We need this table since it will give us the sheet music based on the current concert and part for the band member. This information will be for band members.

  • part_id(VARCHAR) - PK
  • instrument_part(VARCHAR)
  • pdf_url(VARCHAR)

The third database table will store past performances of the concert band. It will store URLs to MP4s or uploaded YouTube links to past performances. This information will be for band members.

  • concert_id(VARCHAR) - PK
  • concert_url(VARCHAR)

The fourth database table will store information about each member of the ensemble and will include a picture and a basic description. This information will be public.

  • member_id(VARCHAR) - PK
  • photo_url(VARCHAR)
  • description(TEXT)

The fifth database table will store information about any upcoming concerts for the general public.

  • concert_id(VARCHAR) - PK
  • description(TEXT)
