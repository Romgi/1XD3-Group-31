Project Overview

Project Name: McMaster Concert Band Website & Music Library

Client: McMaster Concert Band

The McMaster Concert Band is a mixed ensemble of approximately 70 members. The goal of this project is to design and develop a modern, McMaster-branded website that serves both as a public-facing information hub and an internal platform for music distribution and band resources.

Target Users

The website is designed to serve the following primary user groups:

  • Current Band Members: Require streamlined access to music parts, reference recordings, and accurate season information.
  • General Public / Concertgoers: Seek information about the ensemble, upcoming concerts, past performances, and recordings.

Static Information

The website includes the following static content to meet the client’s informational needs:

  • Home page with an overview and description of the ensemble
  • Musicians page listing members by section and instrument
  • Conductor biography
  • Current season schedule with upcoming concerts and performances
  • Past concerts archive including programmes
  • Videos section featuring embedded YouTube recordings

Design Elements

The visual design of the website aligns with McMaster University branding and emphasizes a clean, modern aesthetic. Design considerations include:

  • Use of McMaster colors, logos, and branding where appropriate
  • Consistent layout and navigation across all pages
  • Clear typography and structured content hierarchy
  • Responsive design for mobile and desktop devices

Data

The project manages multiple types of data to support its functionality:

  • User accounts with role-based access (administrator and band member)
  • Music part files uploaded by administrators
  • Instrument tags associated with each music part
  • External reference audio links (e.g., YouTube, Spotify)
  • Concert metadata including dates, titles, and programmes

Interactive Elements

The website includes interactive features designed to improve usability and efficiency:

  • User authentication system with administrator and member roles
  • Music Library page accessible only after login
  • Administrator tools for uploading music and tagging instruments
  • Ability to attach reference audio links to music parts
  • Instrument-based filtering dropdown for band members
  • Embedded video playback for public viewing

Maintenance & Future Use

The website will be maintained by the client and executive team. It will be built using a flexible, code-based approach primarily relying on HTML, CSS, and JavaScript, allowing future teams to update content and extend functionality as needed.

Project Goals & Success Criteria

  • Centralize all band-related information and resources
  • Simplify music distribution for band members
  • Provide a professional public-facing presence
  • Serve as a single hub for schedules, recordings, and ensemble information
