The group is designing a website and app for the McMaster Concert Band. The app is called ConcertHelper.

The Purpose

The project provides a public information hub and an internal platform for music distribution and band resources. It addresses two main needs: current band members need easy access to music parts, reference recordings, and season information, while concertgoers need easy access to ensemble details, upcoming concerts, past performances, and recordings.

Data Description

The app stores several related types of data.

  • Recordings: reference recordings for specific parts and concerts.
  • Parts: uploaded PDF music parts for each concert and instrument.
  • Concerts: upcoming and past performance information.
  • Members: public member information including photo, section, and description.
  • Users: login accounts for administrators and band members.

Functions Performed By App

Admin

  • Add and update concerts.
  • Upload and update music parts.
  • Add reference recordings.
  • Create member profiles.
  • Assign members to parts and recordings.
  • Reset user passwords.

Band User

  • View assigned music parts.
  • Open associated reference recordings.

General User

  • View public member information.
  • View upcoming concerts.
  • Open past performances.

Client Feedback

The client wanted the music library workflow to be easy for the conductor or librarian, asked for simple access to reference recordings, and wanted the members list to be easy to update over time.
