Project Overview

Project Name: McMaster Concert Band Website and Music Library

Client: McMaster Concert Band

The McMaster Concert Band is a mixed ensemble of approximately 70 members. The goal of this project is to design and develop a McMaster-branded website that serves as both a public information hub and an internal platform for music distribution and band resources.

Target Users

  • Current Band Members: Need fast access to music parts, reference recordings, and season information.
  • General Public and Concertgoers: Need information about the ensemble, upcoming concerts, and past performances.

Static Information

  • Home page with an overview and description of the ensemble.
  • Musicians page listing members by section and instrument.
  • Conductor biography.
  • Current season schedule with upcoming concerts.
  • Past concerts archive including programs and recordings.
  • Video content featuring ensemble performances.

Design Elements

  • Use of McMaster colors, logos, and branding where appropriate.
  • Consistent layout and navigation across pages.
  • Clear typography and structured content hierarchy.
  • Responsive design for desktop and mobile devices.

Data

  • User accounts with role-based access for administrators and members.
  • Music part files uploaded by administrators.
  • Reference recording links and uploaded media.
  • Concert metadata including titles, dates, times, and locations.
  • Member profile information and section assignments.

Interactive Elements

  • User authentication for administrator and member roles.
  • Member dashboard access to assigned music parts and recordings.
  • Administrator tools for managing concerts, parts, recordings, members, and passwords.
  • Public viewing of concerts and member information.

Maintenance and Future Use

The website will be maintained by the client and future executive teams. It is built with HTML, CSS, JavaScript, PHP, and MySQL so future contributors can update content and extend the app without rebuilding the project from scratch.

Project Goals and Success Criteria

  • Centralize all band-related information and resources.
  • Simplify music distribution for band members.
  • Provide a professional public-facing presence.
  • Serve as a single hub for schedules, recordings, and ensemble information.
